# RozgarSync – AI‑Powered Gig Marketplace

![Animated Workflow](public/assets/workflow.svg)

## Overview
RozgarSync is a modern, premium‑styled web application built with **Next.js 14** (app router) that connects informal gig workers in Pakistan with AI‑assisted job listings, marketplace features, and a progressive‑web‑app experience. The UI follows a dark‑theme with glass‑morphism, custom gradients, smooth micro‑animations, and full i18n support for English and Urdu.

---

## Architecture Summary

| Layer | Technology | Purpose |
|-------|------------|---------|
| **Front‑end** | Next.js (React 18) + Tailwind CSS | SSR/SSG rendering, fast navigation, responsive layout |
| **State & Context** | Custom React providers (`AuthProvider`, `ThemeProvider`, `LocaleProvider`) | Global authentication, theming, locale handling |
| **Internationalisation** | `next‑intl` + locale JSON files | Server‑side and client‑side translations (English/Urdu) |
| **Backend / API** | Next.js API routes (`src/app/api/*`) | Thin serverless endpoints for Firebase, AI services, map data |
| **Data Store** | Firebase (Auth + Firestore) | User accounts, profiles, gig data |
| **AI Services** | Google Generative AI (and optional OpenAI) | Smart suggestions, chatbot, resume analysis |
| **Maps** | Google Maps JavaScript API (`@react-google-maps/api`) | Show gig locations, coverage maps |
| **PWA** | `next‑pwa` plugin | Offline caching, service‑worker registration |
| **Build & Dev** | Turbopack (enabled via `next dev --turbo`) | Ultra‑fast incremental builds in development |

## Core Features
- **Landing page** with animated hero, badge, and glass‑morphism header.
- **Gig marketplace** (`/gigs`, `/gigs/[id]`) with filtering, details, and apply flow.
- **Wallet & Profile** pages for balance, personal info, and skill management.
- **Safety center**, **Onboarding**, and **Demo** showcase pages.
- **Loading skeletons** (`src/app/[locale]/loading.tsx` & `LoadingSpinner`) for instant UI feedback.
- **Dynamic imports** (e.g., `MapView`) to keep initial bundle size low.
- **Responsive RTL support** for Urdu locale.

## Integration Details
1. **Turbopack Enablement** – Modified `package.json` dev script to `next dev --turbo` and added `experimental.turbopack` flag in `next.config.mjs`.
2. **Loading Skeletons** – Added `src/app/[locale]/loading.tsx` and a reusable `LoadingSpinner` component.
3. **Header/Footer Visibility** – Splash screen now hides global `Header`/`Footer` via a `useEffect` hook.
4. **Performance Fixes** – Adjusted `setTimeout` delays, added top padding (`pt‑24 / md:pt‑28`) to all pages to avoid overlap with the fixed header.
5. **Dynamic Map Import** – Wrapped `MapView` with `next/dynamic` to lazy‑load Google Maps only on client side.
6. **PWA Caching** – Configured runtime caching for Google fonts, static images, Firebase storage, and map tiles.
7. **Build Clean‑up** – Implemented scripts to delete `.next` and `node_modules`, reinstall dependencies, and run a fresh production build.

## Getting Started
```bash
# Clone the repo (already in your workspace)
cd "d:/Hackton Project"

# Install dependencies
npm install

# Development server (fast Turbopack builds)
npm run dev   # runs `next dev --turbo`

# Production build & start
npm run build
npm start
```
Open <http://localhost:3000> in a browser. The app works in both English (`/en`) and Urdu (`/ur`).
